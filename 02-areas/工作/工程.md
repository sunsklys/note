## 项目布局

[https://github.com/golang-standards/project-layout/tree/master](https://github.com/golang-standards/project-layout/tree/master)

```bash
.
├── cmd
│   ├── api //api 或 rpc 入口目录
│   │   ├── api // 此文件用于拆分api定义，没有golang文件，忽略
│   │   └── internal
│   │       ├── config // 应用级配置，自动生成，基本不需要修改
│   │       ├── handler // 控制器，自动生成，基本不需要修改
│   │       ├── logic // 逻辑，自动生成（参数验证，DTO转换，biz启动，返回值构建）
│   │       ├── middleware //中间件，自动生成，需要手动补全功能
│   │       ├── svc // 服务，自动生成，全局对象定义，按需修改（此处的定义，不能在biz中使用）
│   │       └── types // 接口定义，自动生成，完全不需要修改
│   └── cli //cli 入口目录
│       └── internal
│           ├── command
│           ├── config
│           ├── logic
│           └── svc
├── call_gen // grpc 客户端自动生成的位置
├── doc // 文档目录，用于自定义文档、swagger.json 等
├── etc
├── internal
│   ├── api // 调用其他http或grpc服务的目录
│   ├── app // 全局配置，自动生成，按需修改
│   ├── biz // 业务，按领域自行拆分，定义entity以及其相关的属性和方法
│   ├── constant // 常量定义
│   ├── dal
│   │   ├── dao //数据访问对象，用于定义业务中使用的sql查询方法，关于sql的操作不能超出包范围
│   │   ├── model // mysql相关模型，自动生成，是dto的一种
│   │   └── query // 数据对象的封装，自动生成
│   ├── dto // 数据传输对象，可以在服务内部自由传递的简单对象（贫血模型|没有方法）
│   └── pkg // 公共方法
├── library
│   └── errcode // 错误码定义
└── tools // 脚手架
```
