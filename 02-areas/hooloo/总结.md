总结  
## hooloo 全部表关系总结

1. 商品与规格（顾客能点什么）

```bash
goods_category ←── goods ──1:N──→ goods_sku（SKU，最小下单单位）
                   │                    │
                   │                    ├── rule_list(JSON) → 指向 spec_rule
                   │                    │
                   ├──1:N──→ goods_tag   │
                   │                    │
spec ──1:N──→ spec_rule ──1:N──→ spec_rule_material ──→ material(原料)
(维度)        (选项)              (选项→原料用量)        咖啡豆/糖浆/牛奶...
杯型           中杯/大杯           中杯→咖啡豆18g
糖度           全糖/半糖/无糖       全糖→糖浆15ml
```

2. 店铺与门店商品（哪家店哪台机器卖什么）

```bash
shop(店铺中枢) ──→ shop_merchant(支付证书)
     │
     ├──1:N──→ shop_goods(店铺上架了哪些商品)
     │              ↓ goods_id
     │         goods
     │
     ├──1:N──→ shop_goods_sku(店铺×机器×SKU 可售/价格/库存)
     │              ↓ sku_id
     │         goods_sku
     │
     ├──1:N──→ shop_recommend(推荐位)
     ├──1:N──→ shop_users(顾客-店铺绑定)
     ├──1:N──→ warehouse_shop(仓库-店铺)
     └──1:N──→ sys_user_shop(后台权限桥)
```

3. 机器与组件（怎么做）

```bash
machine_model(机型)                              machine(SaaS库)
     │                                                ↓ shop_id
     ├──1:N──→ component(模板,表名model_component)     │
     │              ↓ model_id                         ├──1:N──→ machine_component(实例)
     └──1:N──→ component_property(模板)                │              ↓ com_ros_code+pro_ros_code 匹配
                       ↓ material_id(跨库)             ├──1:N──→ machine_component_property(余量值)
                  material(SaaS库)                     │
                                                       ├── machine_config(冰满/延迟等参数)
                                                       ├── machine_device(IoT凭证)
                                                       ├── coffee_machine / coffee_product→coffee_parameter
                                                       ├── machine_warn_alarm(预警阈值)
                                                       ├── material_monitor→material_monitor_op
                                                       └── machine_log / device_log / control_panel_operation
```

4. 订单（从下单到制作完成）

```bash
customer ──→ order(SaaS) ──1:N──→ order_detail(SaaS)
                 │                      │
                 │                 ├── order_refund(退款)
                 │                 ├── order_snapshot(已废弃)
                 │                 ├── order_printer(打印)
                 │                 └── order_operation_log(操作日志)
                 │
                 ↓ 支付后同步
            order(Master) ──1:N──→ order_detail(Master)
                 │                      │
                 │                 ├── order_detail_full(奶量偏移)
                 │                 ├── order_detail_space(仓位)
                 │                 ├── user_order_multi(多杯桥)
                 │                 │
                 │                 ↓ 拆单
                 │            order_risc(制作指令)
                 │                 ↓ 展开
                 │            ros_risc_flow(ROS执行流)
                 │
                 └── coupon_id → coupon_user
```

5. RISC 制作指令流（四级流转）

```bash
risc(指令字典)
  ↑ risc_id
sku_model_risc(机型模板)          ← 建商品时生成
  ↓ 店铺上架实例化
sku_machine_risc(店铺×机器实例)    ← sku/shop/machine/material 全跨库
  ↓ 下单拷贝
order_risc(订单行指令)            ← material_id+quantity 来自 spec_rule_material
  ↓ 拆分展开(1条→多条ROS步骤)
ros_risc_flow(ROS执行流水)        ← start/end 时间，state 0→1→2，可重试4
```

6. 优惠券与活动（怎么省钱）

```bash
coupon_category(类别)                coupon_packages(券包)
  ↓ category_id                        ↓ coupon_cfg(JSON[{coupon_id,num}])
coupon(优惠券)                         ↓
  ├── rule_id → coupon_rule(规则)      ↓ 购买后发券
  └── ←────────────────────────── coupon_user(用户领取记录)
                                        ├── cust_id → customer
                                        ├── order_id → order(使用时回填)
                                        └── source = activity/coupon_package/poster_activity/...

hooloo_activities(首页活动)              poster_activities(海报活动)
  ↓ activity_type: 首页/支付成功/新人送    ↓ activities_type: 发券/抽奖/分享/邀请
home_activity_users(参与记录)             poster_activity_users(参与记录,防重靠它)
                                          poster_activity_shared_users(裂变,下单时发券)
```

7. 库存与仓库（物料账）

```bash
warehouse(总仓) → warehouse_shop(店铺即二级仓) → warehouse_merchandise(货品库存余量)
                                                      ↓ ware_mer_id
                                                 ├── warehouse_merchandise_log(出入库+清点流水)
                                                 ├── warehouse_merchandise_count(清点)
                                                 ├── warehouse_merchandise_loss(报损)
                                                 ├── warehouse_merchandise_usage(按订单消耗)
                                                 ├── warehouse_merchandise_usage_day(按天汇总)
                                                 └── warehouse_merchandise_material(原料-货品绑定)

merchandise_info(货品字典) → material(原料)
warehouse_material_fake(真实↔虚拟原料映射)
archievement_cost / archievement_revenue(月度绩效)
```
